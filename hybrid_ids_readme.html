Architecture

XGBoost Classifier
100 estimators · learning rate 0.1 · max depth 6. Gradient boosting for strong predictive accuracy on tabular traffic data.
LightGBM Classifier
100 estimators · learning rate 0.1 · max depth 6. Leaf-wise tree growth for speed without sacrificing accuracy.
Soft Voting Ensemble
Averages class probabilities from both models. Reduces individual model bias and improves generalisation.

FAQ

What attack types can it detect?
The model is trained on NSL-KDD categories: DoS (Denial of Service), Probe, R2L (Remote to Local), U2R (User to Root), and Normal traffic. It uses a multi-class label encoder to classify each network connection.
Why hybrid XGBoost + LightGBM?
Both are gradient boosting algorithms but use different tree-building strategies. XGBoost uses level-wise growth (more stable), while LightGBM uses leaf-wise growth (faster, higher accuracy on large data). Combining them via soft voting reduces variance and improves generalisation.
What CSV format should I upload?
Your CSV must have the same feature columns as the NSL-KDD dataset (minus attack_type and difficulty_level). The app automatically one-hot encodes categorical features and aligns columns to the trained model's feature space.
Model file not found error?
Run python train_model.py first to generate hybrid_artifacts.joblib. This file must be in the same directory as app.py before launching Streamlit.
